                           ROOT CAUSE SUMMARY
================================================================================

USER EXPECTATION:
  "MLflow should be tracking my LLM calls"

ACTUAL STATE:
  MLflow is DISABLED because:
    1. PROMPTCHAIN_MLFLOW_ENABLED environment variable NOT set (defaults False)
    2. MLflow library NOT installed (pip install mlflow)
    3. Ghost pattern correctly disables decorators when tracking disabled

EVIDENCE:
  ✓ Diagnostic confirms _ENABLED = False
  ✓ run_model_async has no closure (decorator is identity function)
  ✓ init_mlflow() exits early at line 678 (disabled check)
  ✓ Callback system works (6 events captured)
  ✓ MLflow UI shows 0/0 activities

WHY ZERO TRACKING:
  The ghost pattern is working AS DESIGNED. When MLflow is disabled:
    - conditional_decorator() returns identity function
    - Decorators become transparent (zero overhead)
    - No MLflow runs created
    - No HTTP calls to tracking server

  This is NOT a bug - it's a feature. The ghost pattern ensures:
    - Zero performance penalty when disabled
    - No dependencies when not needed
    - Clean separation of concerns

  User issue: Expected MLflow to be enabled by default, but it's opt-in.
